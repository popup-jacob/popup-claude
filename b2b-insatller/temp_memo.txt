ADW Installer 방향성 검토 및 MCP Gateway 제안
================================================

1. 현재까지의 경과
------------------

ADW Installer는 일반 사용자가 AI-Driven Work 환경을 쉽게 구축할 수 있도록 만든 도구입니다.
랜딩페이지(adw.popupstudio.ai)에서 원하는 서비스를 선택하면 원클릭 설치 명령어가 생성되고,
현재 모듈로 Google Workspace, Atlassian이 있고 앞으로 Slack, Notion, GitHub, Figma 같이 여러 서비스를 지원할 예정입니다.


2. B2B 전환 시 핵심 과제
-------------------------

bkit이 고도화되면서 B2B 사업으로 연결하는 것이 목표가 되었고,
고객사 직원들이 쉽게 ADW 환경을 구축할 수 있는 페이지(start.bkit.ai)가 필요한 상황입니다.

그런데 기업을 상대로 할 때 가장 중요한 것은 세세한 접근 범위(Scope) 커스터마이징입니다.
어떤 직원이 어떤 서비스의 어떤 데이터까지 AI로 접근할 수 있는지를 통제할 수 있어야 합니다.


3. 현재 MCP 기술의 딜레마
--------------------------

MCP 생태계가 계속 고도화되고 있는 상황에서, 두 가지 방식 중 선택이 필요합니다.

  [Local MCP - 현재 방식]
    - 사용자 PC에 직접 MCP 서버를 설치하는 방식
    - Scope 설정 가능 (AI가 접근하는 범위를 세부적으로 제한할 수 있음)
    - 단, 토큰 발급 등 사람이 직접 개입해야 하는 과정이 반드시 존재함

  [Remote MCP - 새로운 흐름]
    - OAuth 기반으로 클라우드에서 연결하는 방식
    - 설정이 매우 간편함 (클릭 몇 번으로 연결 완료)
    - 단, Scope 설정이 불가능 (연결하면 서비스 전체에 대한 접근 권한이 부여됨)

  [고려사항 1]
    기업 상대로는 Scope 커스터마이징이 필수적인데,
    Scope 설정이 안 되는 Remote MCP를 선택하는 것이 맞는가?

  [고려사항 2]
    그렇다고 Local MCP를 선택하면, 아무리 자동화를 해도 토큰 발급처럼
    사람이 개입해야 하는 과정이 생기고,
    고객사 직원들이 이를 따라가기 어려워 우리 인력이 온보딩 지원에 소모될 가능성이 있음.


4. 해결 방안: MCP Gateway
--------------------------

위 두 가지 문제를 동시에 해결할 수 있는 방법이 MCP Gateway입니다.

  - Remote MCP 방식을 사용하면서도, Gateway 레벨에서 자체적으로 Scope를 설정할 수 있음
  - 고객사 직원들은 회사의 MCP Gateway에 연결하기만 하면 되므로 설정이 간단함
  - 추가로 접근 로그, 토큰 관리, 사용량 등의 모니터링이 가능해짐


5. bkend 활용 시 추가 이점
---------------------------

이미 보유하고 있는 bkend 플랫폼 위에 MCP Gateway를 구축하게 되면,
위의 모든 기능을 포함하면서 다음이 추가로 가능합니다.

  - bkend 콘솔에서 관리 UI 제공 (서비스 연결, 역할/Scope 설정, 직원 관리)
  - 사용량 통계를 그래프, 상세 차트 등으로 시각화
  - 기존 bkend의 인증(Auth), 데이터베이스(DB), API 기능 재활용
  - B2B SaaS 구독 모델로 수익화 가능


6. ADW Installer 방향성 재설정
-------------------------------

위 내용을 종합했을 때, 현재 ADW Installer의 방향성을 재설정해야 한다고 생각합니다.

현재는 사용자가 편하게 설치하는 것에 집중하여, 헷갈리거나 고민될 부분은 자동으로 처리했습니다.
하지만 기업을 상대로 한다면, 설치 시 선택할 수 있는 옵션을 더 세분화하여
고객사의 환경과 요구사항에 맞게 커스터마이징할 수 있는 구조가 필요합니다.
